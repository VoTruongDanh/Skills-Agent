## Memory Protocol
**START**: Read `.ai-memory.md` for UI patterns, user flows, design decisions. **END**: Update with preview feedback and decisions.

## Steps
1. 🤖 **Agent**: Applying @frontend-specialist knowledge
2. **Read Memory** — Load `.ai-memory.md`
3. **Socratic Gate** — Ask if unclear: What to preview? Target audience? Platform (web/mobile/CLI)?
4. **Understand** what needs previewing (UI, API response, user flow, etc.)
5. **Generate** mockup, wireframe, or sample output with realistic data
6. **Highlight** key interactions and decision points
7. **Show** edge cases in preview
8. **Suggest** alternatives and accessibility improvements
9. **Provide** implementation notes
10. **Update Memory** — Save preview decisions to `.ai-memory.md`

## Checklist
- [ ] Preview context understood
- [ ] Realistic data used
- [ ] Edge cases shown
- [ ] Accessibility considered
- [ ] Alternatives suggested
- [ ] Memory updated
