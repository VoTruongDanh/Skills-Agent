## Memory Protocol
**START**: Read `.ai-memory.md` for design system, UI patterns, past feedback. **END**: Update with UI decisions and design patterns used.

## Steps
1. 🤖 **Agent**: Applying @frontend-specialist knowledge
2. **Read Memory** — Load `.ai-memory.md`
3. **Socratic Gate** — Ask if unclear: Target platform? Design system in use? Accessibility requirements?
4. **Review** current UI/UX implementation
5. **Identify** usability issues, visual inconsistencies, accessibility gaps
6. **Suggest** specific improvements with rationale
7. **Provide** mockups, wireframes, or code examples
8. **Consider** WCAG accessibility, responsive design, mobile/desktop
9. **Refine** copy for clarity and user needs
10. **Update Memory** — Save design decisions to `.ai-memory.md`

## Checklist
- [ ] Current UI reviewed
- [ ] Usability issues identified
- [ ] Improvements are incremental (not full redesign)
- [ ] Accessibility (WCAG) considered
- [ ] Responsive design addressed
- [ ] Copy is clear and concise
- [ ] Memory updated
